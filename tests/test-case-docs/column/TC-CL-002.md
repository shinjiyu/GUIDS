# Column 组件对齐属性测试

## 测试目的

验证 Column 组件的对齐属性（alignment）功能，确保组件能够正确应用不同的对齐方式。

## 测试环境

- GUIDS2HTML 转换器版本：1.0.0
- 支持的浏览器：Chrome 90+、Firefox 88+、Safari 14+

## 前置条件

1. GUIDS2HTML 转换器已正确安装和配置
2. 测试数据文件 `TC-CL-002.json` 存在且格式正确

## 测试步骤

1. 加载测试数据文件 `TC-CL-002.json`
2. 使用 GUIDS2HTML 转换器将 JSON 转换为 HTML
3. 在支持的浏览器中打开生成的 HTML 文件
4. 观察不同对齐方式下 Column 组件的渲染结果

## 测试数据

测试数据包含四个 Column 组件，分别使用不同的对齐方式：

1. start：子元素靠左对齐
2. center：子元素居中对齐
3. end：子元素靠右对齐
4. stretch：子元素宽度拉伸至容器宽度

## 预期结果

1. start 对齐的 Column：

   - 子元素靠左对齐
   - 子元素保持原有宽度
   - 子元素垂直排列

2. center 对齐的 Column：

   - 子元素居中对齐
   - 子元素保持原有宽度
   - 子元素垂直排列

3. end 对齐的 Column：

   - 子元素靠右对齐
   - 子元素保持原有宽度
   - 子元素垂直排列

4. stretch 对齐的 Column：
   - 子元素宽度拉伸至容器宽度
   - 子元素垂直排列
   - 子元素之间保持间距

## 验证点

1. 每个 Column 组件是否正确应用了指定的对齐方式
2. 子元素的位置是否符合对齐方式的要求
3. 子元素的尺寸是否正确（特别是 stretch 模式下的宽度）
4. 子元素之间的间距是否正确
5. 整体布局是否美观且符合预期
